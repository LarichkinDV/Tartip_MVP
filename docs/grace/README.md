# GRACE Artifacts

This directory stores lightweight project governance artifacts used before
changing domain logic or infrastructure.

- `knowledge-graph.xml`: domain entities, allowed edges, and forbidden direct edges.
- `module-contracts.xml`: module responsibilities, inputs, outputs, and invariants.
- `verification-plan.xml`: verification scenarios for governance and versioning rules.
- `execution-packets.xml`: planned and ready execution packets such as
  `EP-001-INFRA`, `EP-002-REFERENCE-GOVERNANCE`,
  `EP-003-REFERENCE-VERSIONING`,
  `EP-004-PROJECT-PLANNING-AND-ACCEPTANCE`, and
  `EP-005-ACCEPTANCE-AND-USER-ACTION-DASHBOARDS`, and
  `EP-007-VERIFICATION-DASHBOARD`, and
  `EP-008-DISSERTATION-PROMPT-GENERATION`, and
  `EP-009-CODEX-SPEC-AUDIT`, and
  `EP-011-GIT-WORKFLOW-DISCIPLINE`.

For `EP-001-INFRA`, Docker-dependent checks are allowed to be skipped locally when
Docker is unavailable, but the skip must be explicit. Non-Docker checks still run
through `make format`, `make lint`, `make test`, `make check`, and the reference
validation commands where applicable.

For `EP-005-ACCEPTANCE-AND-USER-ACTION-DASHBOARDS`, local dashboards are
generated by `make generate-dashboards` and validated by `make validate-plan`.
They are Markdown/YAML artifacts only; no dashboard web UI, frontend route, or
backend endpoint is part of this packet.

For `EP-007-VERIFICATION-DASHBOARD`, the manual verification dashboard is
generated by `make generate-verification-dashboard` and validated by
`make validate-verification`. It does not replace acceptance dashboards or the
monthly test protocol.

For `EP-008-DISSERTATION-PROMPT-GENERATION`, dissertation synchronization is
limited to impact logs, prompt queues, markdown patch queues, prompt profiles,
and validation scripts. It must not edit DOCX directly, create a new DOCX
version, invent bibliography sources, or mark dissertation updates accepted.

For `EP-009-CODEX-SPEC-AUDIT`, audit is an audit-first/read-mostly contour.
It creates audit reports, language findings, and validation scripts, but does
not mass-russify existing documents or modify accepted/protected artifacts
without user approval.

For `EP-011-GIT-WORKFLOW-DISCIPLINE`, Git validation is advisory by default.
It checks branch naming, dirty working tree scope, merge acceptance gates, and
forbidden files, but does not run git add, commit, push, merge, or branch
deletion.
