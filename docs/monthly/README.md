# Месячное планирование

## 1. Назначение

Месячное планирование задает отчетно-защитную структуру развития прикладного прототипа ТАРТИП.
Оно связывает технические execution packets с документами, которые можно показывать руководителю или Заказчику как проверяемые результаты месяца.

## 2. Правило месячного блока

Месяц является тематическим блоком.
Каждый месячный блок содержит 3 задачи.
Каждая задача планируется как 15 человеко-часов.
Плановый объем месяца составляет 45 человеко-часов.

Результат месяца:

- БФТ;
- ТЗ;
- протокол испытаний.

## 3. Разграничение контуров

Execution packets описывают техническую реализацию, изменения файлов и проверки в репозитории.
Monthly blocks описывают отчетно-защитную структуру месяца.
БФТ, ТЗ и протокол испытаний являются защищаемыми документами месяца.

## 4. Prompt templates и итоговые документы

Prompt templates могут использоваться как supporting artifacts для подготовки материалов.
Итоговые БФТ, ТЗ и протокол испытаний являются source artifacts.
Prompt не заменяет БФТ, ТЗ или протокол испытаний.

## 5. Приемка

Codex готовит документы, проверки и сводки.
Пользователь принимает документы и результаты.
После пользовательской приемки accepted monthly documents становятся protected artifacts и не должны изменяться без explicit user approval.

## 6. Customer-Facing Principle

Месячные документы пишутся как документы для руководителя или Заказчика.
Они описывают прикладной прототип, локальное развертывание, модульное развитие, проверяемость и трассируемость.

Месячные документы не раскрывают личную стратегию коммерциализации.
Они не заявляют автоматический сбор данных Заказчика.
Они фиксируют только прикладной прототип и его проверяемые результаты.
